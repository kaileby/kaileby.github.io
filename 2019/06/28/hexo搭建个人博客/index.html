搭建博客

发表于 2019-06-28 更新于 2020-06-07 分类于 博客相关

hexo个人博客搭建

环境准备

  • Node.js

    node -v npm-v

    测试 Node.js 是否安装成功

  • cnpm

    npm install -g cnpm --registry=https://registry.npm.taobao.org

    安装好了就可以使用cnpm来替代npm进行js依赖的下载了

  • hexo

    cnpm install -g hexo-cli

    使用 cnpm 安装 hexo 框架；hexo -v 验证 hexo 是否安装成功

开始搭建

  • 创建文件夹作为博客的仓库

    在命令行窗口中切换到新建好的博客仓库的当前目录下；

  • 初始化

    hexo init

    执行该命令后，会在 博客仓库 下，生成一系列文件 ，使用dir /a *可在 cmd 窗口中查看所有生成的文件

  • 启动博客

    hexo s

    之后，命令窗口会提示在 http://localhost:4000查看本地的博客，至此个人博客就已经搭建成功了。

    在发布、部署到github之前，可以在本地进行预览，ps：此时还不能再网站中查看到内容。

    • 可以试着在 博客仓库下的 source 文件夹下新建一个 md 文件，并在浏览器预览。

  • 导入资源

    D:\Project\myBlog\source\_posts

    myBlog 是个人仓库名，资源都是放在source\ _posts目录下的；在搭建环境之后，就可以往里面放置自己的md 文档

  • 将新增加的内容与博客建立关联

    hexo g 相当于刷新的作用，用于同步 myBlog 中的任何改动，譬如：博客内容的删减、博客样式的变动等。

    然后 hexo s 重启一下，在浏览器中，输入 http://localhost:4000 预览效果

  • 部署到 GitHub

    • 登录 GitHub, 新建一个 Repository

      PS: 仓库名必须是 xxx.github.io 形式,xxx是github的用户名，本人的仓库是 kaileby.github.io

  • 回到命令行，在 博客仓库下安装 Git 插件

    cnpm install --save hexo-deployer-git

    利用 Git 工具，将博客仓库上传到 Github

  • 设置 config.yml 文件

    在文件末尾，大概 80 行的样子，进行上述的修改设置；

    PS：注意 type: 之后空格再写 git，直接写会报错….

  • 部署到远端

    hexo d

    刷新 Github 中新建的那个 Reposity，可以看到多出来许多文件，接下来输入：https://kaileby.github.io/ 即可访问到部署在GitHub上的博客

给 Hexo 换样式

  • 下载 Theme

    tips:切换到D:\Project\myBlog\，再执行下面的命令

    git clone https://github.com/litten/hexo-theme-yilia themes/yilla

    将下载的主题 clone 到 themes下的 yilia 文件夹下

  • 修改 config.yml 文件

    默认 theme: landscape,将 landscape 替换成 yalla ;

    接着，使用 hexo clean , hexo s 即可看到效果

遇到的问题

  • 安装 Node.js 没有 npm

    卸载 Node.js 以及 npm在 Appdata下创建的两个文件夹，重启，再重装一次解决了…有点莫名其妙！

发布资源时遇到的问题

  • 发布资源

    • 进入 cmd ，以 D:\Project\myBlog 为当前目录
    • 将新的博客内容放置在 source\ _posts 目录下，hexo g同步到博客的最新状态；
    • hexo d，将博客的新状态部署到github，等几秒，即可在网站中见到更新的的内容

  • 文章中存在嵌套引用

    • 在 Typro 中时完全支持多层嵌套的，但对于 hexo,它只接受一层引用。

    • 具体表现：hexo g 时报错

      ValidationError: source is required!

    • 解决方法：将文章中的引用全部改成一层

  • 内嵌的 HTML 语句也会报ValidationError: source is required!错
